Voice

Voice & Tone

Professional-casual. Technically credible with personality. Lead with the concrete number, not the explanation. Helpful in errors, playful in marketing.

Do

"Like battery percentage, but for your gains."

Don't

"A comprehensive portfolio analytics dashboard solution for active traders."

Do

"Your Schwab token has expired. Re-authenticate in Terminal:" + copyable command

Don't

"We're sorry, an unexpected error occurred. Please contact support."

Do

"Forever (or until my AWS bill becomes sentient)"

Don't

"Your data is stored securely according to industry best practices."

Patterns

Design Principles

Eight rules that guide every design decision across the product.

01

Numbers first

The P&L value is always the most prominent element — large, monospaced, colored.

02

Green means up

Green is the only brand color. It signals gains, primary CTAs, and the $ prompt.

03

Native on native

The macOS app uses SF fonts, system colors, and standard materials. No custom chrome.

04

Progressive disclosure

Timeframes → tickers → executions → matched opens. Each click reveals one more layer.

05

Stale beats empty

During refresh, show the previous value. Never flash to a loading state after first load.

06

Monospace for money

All financial numbers use monospaced fonts with tabular numerals for column alignment.

07

Dark by default

The landing page commits to dark. The native app defers to macOS appearance.

08

Functional color only

Green = gain. Red = loss. Amber = warning. No decorative color use anywhere.
